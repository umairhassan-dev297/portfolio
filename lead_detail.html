Lead Management System

A role-based case management system built with Laravel — admins upload bulk cases via Excel, assign them to agents, track follow-ups, and manage everything through a calendar integrated dashboard.

Built a complete Lead Management System at WebCrazier using Laravel and MySQL. The system supports two roles — Admin and Agent. Admins can upload hundreds of cases at once using Excel files, assign them to agents, track follow-ups, and manage case statuses. Agents handle their assigned cases and update progress. A calendar view helps the admin track all case assignment dates at a glance — making operations much more organized and efficient.

The main challenge was handling bulk Excel uploads efficiently — parsing hundreds of rows at once, validating the data, and saving it cleanly to the database without errors. On top of that, the system needed a calendar that could visually show which cases were assigned on which dates, while also keeping the admin and agent roles strictly separated with their own dashboards and permissions.

Used Laravel Excel package to handle bulk imports — it reads the uploaded Excel file, validates each row, and stores the cases in one clean operation. Calendar integration was done to display case assignment dates visually so the admin can easily track workload. Laravel's RBAC was used to separate Admin and Agent access, and case statuses like "In Progress", "Completed", and "Cancelled" were managed with a simple status system.

Key Features

  • Bulk Case Upload via Excel
  • Case Assignment to Agents
  • Calendar — Track Assignment Dates
  • Case Edit, Delete & Follow-up
  • Role-based Access — Admin & Agent
  • Case Status Management
